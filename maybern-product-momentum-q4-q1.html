MXL: Expand the Calculation Engine

MXL is Maybern's proprietary expression language that lets clients define custom calculations across every dimension of their data. In the past six months we've dramatically expanded what's possible — delivering real-time feedback that lowers the time and cost to insight, and new platform flexibility that broadens MXL's reach across the system.

Real-Time Feedback

Increased real-time feedback to lower time and cost to insight. Users now get instant signals as they work — no more save-and-check cycles or support tickets for calculation authoring.

Platform Flexibility

Expanding MXL's reach across the platform to power calculations wherever clients need them. Metrics that used to be hardcoded are now fully composable within MXL.

MXL Autocomplete

As users type MXL expressions, the system now provides intelligent suggestions — surfacing available functions, field names, and syntax patterns in real time. This dramatically lowers the barrier to writing correct expressions on the first try and reduces the need for support-assisted configuration.

Shipped Real-Time Feedback
MXL Live Evaluation

Users can now see the results of their MXL expressions in real time as they write them. Live evaluation provides an instant feedback loop — turning expression authoring into an interactive, exploratory experience instead of a blind save-and-check workflow.

Shipped Real-Time Feedback
Calculations & Expressions Engine

A foundational overhaul of the underlying calculation layer that powers all MXL expressions. This upgrade ensures consistent, performant evaluation across the entire platform — from report columns to allocation rules to performance metrics.

Shipped Platform Flexibility
MXL In-App Documentation

Comprehensive documentation for every MXL function and syntax pattern is now surfaced directly within the application. Users no longer need to leave the workflow to look up references — everything is contextual and searchable.

Shipped Real-Time Feedback
Credit Facility Metrics → MXL Migration

Credit facility metrics — including drawn amounts, availability, and utilization ratios — have been migrated to the MXL framework. Clients can now customize, extend, and compose credit facility calculations with the same flexibility as any other MXL expression. This is the first of many domain-specific metric sets moving onto MXL.

Shipped Platform Flexibility

Report Builder — build, configure, and run MXL-powered performance reports
How It Works

Report Builder is built on MXL — Maybern's expression language. Every column in a report is an MXL calculation, which means users can build reports that run custom performance calculations at any granularity (from individual investment up to LP) instantly. Choose between Snapshot Reports (one "as of" date) and "Over Time" Reports (calculations across each increment of a date range). Add entity-level metadata via Attributes, apply filters, and save the whole configuration as a reusable template that anyone on the team can run.

Core Accounting Power

The foundation of Maybern is accurate, automated fund accounting. Over the past six months we've shipped significant enhancements that expand the depth and flexibility of the core ledger and allocation engine — including first-class support for Alternative Investment Vehicles and Cashless Contributions.

01 Alternative Investment Vehicle (AIV) Support

Maybern now supports full lifecycle management for AIVs — deal-specific vehicles created for tax, regulatory, or structuring purposes. The AIV release delivers comprehensive support across the entire accounting workflow: entity configuration and allocation graph integration, capital call writes, capital activity views, distribution processing, invested period fees, waterfall calculations, edit rules and closings, credit facility integration, adjustments, and transfers. AIVs are treated as first-class entities within the Maybern allocation framework, inheriting the same depth of calculation and auditability as any other fund structure.

Shipped
02 Cashless Contributions

Full support for cashless contribution workflows — where a GP's capital contribution is funded not by cash but by LP fee waivers. Maybern handles both the GP Priority Distribution and Optional Clawback methodologies end-to-end, including special profits allocations, fee waiver capital calls, distribution processing, and hypothetical waterfall calculations. This eliminates a complex, error-prone process that clients previously managed entirely in spreadsheets.

Shipped
03

Bulk Cash Flow Ingestion

A new high-throughput engine for ingesting large batches of cash flow events — capital calls, distributions, and transfers — in a single operation. Designed for onboarding and period-end processing at scale.

Shipped
04

Event-Level Overrides

Users can now apply targeted overrides at the individual cash flow event level — providing surgical precision for handling exceptions, corrections, and client-specific adjustments without rebuilding entire transaction sets.

Shipped
05

Named Dates

User-defined reference points like "Fund Close Date" or "Audit Period End" that can be used across MXL expressions, reports, and allocation logic. Reduces hardcoded dates and makes configurations more readable.

Shipped
06

Workflow Primitives

The first building blocks of Maybern's workflow engine — supporting waterfall calculations and scenario modeling. Clients can define multi-step distribution waterfalls and model "what-if" scenarios before committing.

Shipped
Why This Matters

AIVs and Cashless Contributions represent two of the most structurally complex workflows that legacy systems aren't equipped to handle. Maybern handles them natively — with full auditability, allocation graph integration, and MXL-powered calculations throughout.
